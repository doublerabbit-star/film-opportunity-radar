# Film Opportunity Radar - Design Brief (v0.1)

## Product Position

Film Opportunity Radar is not a movie database or a news website.

It is a research tool for film creators that helps them quickly discover today's best content opportunities.

The UI should feel like an editorial publication rather than a SaaS dashboard.

---

# Design Keywords

- Editorial
- Magazine
- Archive
- Cinema
- Clean
- Modern
- Playful
- Research Tool

Avoid:

- Generic Tailwind Dashboard
- Enterprise SaaS
- Heavy Glassmorphism
- Tech Startup Style

---

# Visual Direction

The overall feeling should resemble:

- an independent film magazine
- a creative archive
- a design publication

The interface should feel carefully designed but never decorative for its own sake.

Whitespace is important.

Typography is more important than decoration.

Movie posters are the visual focus.

---

# Layout

The application contains three core pages:

- Home (Today's Opportunities)
- Opportunity Detail
- Watchlist

Use a fixed left sidebar for navigation.

The main content should use a magazine-like grid with generous spacing.

Avoid KPI dashboards and complex admin layouts.

---

# Typography

Use a strong editorial headline style.

Large bold headings are encouraged.

Body text should remain clean and readable.

Create a clear hierarchy:

- Hero Heading
- Section Heading
- Card Title
- Body
- Caption

---

# Components

The most important reusable component is the Opportunity Card.

Each card should contain:

- Poster
- Event Type
- Title
- Short Description
- Opportunity Level
- Source
- Published Time
- Bookmark Button

Other reusable components:

- Sidebar
- Filter Tabs
- Opportunity Badge
- Search
- Empty State

---

# Interaction

Interactions should feel subtle and tactile.

Inspired by A24.

Examples:

- Poster slightly scales on hover.
- Text has a subtle movement on hover.
- Filter tabs slide naturally.
- Bookmark interaction feels like saving an archive item.

Animations should support the interface rather than attract attention.
